# InfinityTrace Quick Start Guide

## Installation

```bash
# Clone the repository
git clone https://github.com/Atikulislamx/InfinityTrace.git
cd InfinityTrace

# Install dependencies
pip install -r requirements.txt
```

## Basic Usage

### 1. Simple Username Search
```bash
python infinitytrace.py --username john_doe
```

### 2. Multi-Input Analysis
```bash
python infinitytrace.py --username john_doe --email john@example.com --phone +1234567890
```

### 3. With JSON Export
```bash
python infinitytrace.py --username john_doe --json
```

### 4. Custom Output File
```bash
python infinitytrace.py --username john_doe --output my_report.txt
```

### 5. Full Analysis
```bash
python infinitytrace.py --username john_doe --email john@example.com --phone +1234567890 --name "John Doe" --output report.txt --json
```

## CLI Options

| Option | Description | Example |
|--------|-------------|---------|
| `--username` | Username to search | `--username john_doe` |
| `--email` | Email to analyze (public only) | `--email john@example.com` |
| `--phone` | Phone to analyze (public only) | `--phone +1234567890` |
| `--name` | Full name for soft search | `--name "John Doe"` |
| `--output` | Output filename (default: output.txt) | `--output report.txt` |
| `--json` | Export JSON format | `--json` |
| `--help` | Show help message | `--help` |

## Output Files

### Text Report (output.txt)
Human-readable report with:
- Target summary
- Platforms found
- Risk indicators
- Email/phone checks
- Final assessment with risk score
- Disclaimer

### JSON Report (output.json)
Machine-readable JSON with:
- Metadata
- Input details
- Analysis results
- Risk assessment

## Understanding Risk Scores

Risk scores range from 0-100 with three levels:
- **LOW (0-30)**: Minimal risk indicators detected
- **MEDIUM (31-60)**: Moderate risk indicators present
- **HIGH (61-100)**: Multiple risk indicators detected

### Risk Score Components (Weighted)
- Username reuse: 20%
- Profile behavior patterns: 25%
- Image reuse: 15%
- Domain reputation: 25%
- Language manipulation: 15%

## Troubleshooting

### No platforms found
- Username may not exist on searched platforms
- Network issues may prevent platform checks
- Some platforms may block automated requests

### Low risk score despite multiple inputs
- Risk score is based on detected patterns, not just data quantity
- Multiple clean inputs result in low scores (which is good!)

### JSON file not generated
- Ensure you use the `--json` flag
- Check file permissions in the output directory
